## 1. MVP 対象（5カテゴリ）

以下の 5 カテゴリは E2E テストで必ずカバーする。各カテゴリに対して代表的な入力パターンを 1 つ固定し、Phase 1 でユーザー操作順序を文書化する。

| # | カテゴリ | 代表入力形式 | 参照問題 | 優先度 |
|---|---------|-------------|---------|-------|
| 1 | 基本配列 | `N` / `A_1 A_2 ... A_N` | ABC395-A, ABC360-C | **MVP** |
| 2 | グリッド | `H W` / `S_1` / `...` / `S_H` | ABC390-C | **MVP** |
| 3 | 木入力（辺リスト） | `N` / `u_1 v_1` / `...` / `u_{N-1} v_{N-1}` | 典型 ABC-D グラフ問題 | **MVP** |
| 4 | クエリ列（Choice） | `N Q` / 各クエリ `1 a b` / `2 a b` / `3 a` | ABC395-D, ABC380-E | **MVP** |
| 5 | 複数テストケース | `T` / 各ケース: `N` / `A_1...A_N` | 典型マルチケース | **MVP** |

### 1.1 基本配列

**入力形式**:
```
N
A_1 A_2 ... A_N
```

**制約**:
- 1 ≤ N ≤ 10^6
- 1 ≤ A_i ≤ 10^9

**E2E で検証する操作**:
- scalar 作成（N）
- 横配列作成（A, 長さ N）
- draft range 自動生成
- range 値の入力 → completed constraint 昇格
- 右ペイン: TeX 入力形式 + TeX 制約 + sample 表示

**参照**: ABC395-A（N / A_1...A_N）、ABC360-C（複数独立配列は中級ケース）

### 1.2 グリッド

**入力形式**:
```
H W
S_1
S_2
...
S_H
```

**制約**:
- 1 ≤ H ≤ 500
- 1 ≤ W ≤ 500
- S_i は英小文字からなる長さ W の文字列

**E2E で検証する操作**:
- 同一行 tuple 作成（H, W）
- 文字グリッドテンプレート適用
- |S_i| = W の自動生成
- charset draft → 英小文字に設定
- 右ペイン: TeX グリッド表示 + sample grid 生成

**参照**: ABC390-C（H W / S_1...S_H）

### 1.3 木入力（辺リスト）

**入力形式**:
```
N
u_1 v_1
u_2 v_2
...
u_{N-1} v_{N-1}
```

**制約**:
- 2 ≤ N ≤ 2×10^5
- 1 ≤ u_i, v_i ≤ N
- グラフは木である

**E2E で検証する操作**:
- scalar 作成（N）
- 辺リストテンプレート適用（本数 = N - 1 式）
- 辺 range の自動生成（1 ≤ u_i ≤ N, 1 ≤ v_i ≤ N）
- Property ショートカット: Tree 追加
- 右ペイン: TeX + sample tree 生成

**参照**: 典型 ABC-D 木問題

### 1.4 クエリ列（Choice）

**入力形式**:
```
N Q
1 a b
2 a b
3 a
```
（Q 行のクエリ。先頭の数字でバリアント分岐）

**制約**:
- 1 ≤ N ≤ 2×10^5
- 1 ≤ Q ≤ 2×10^5
- 各バリアントの変数に range 制約

**E2E で検証する操作**:
- 同一行 tuple 作成（N, Q）
- クエリ列テンプレート適用（count = Q）
- variant 3 つ追加（tag=1,2,3）
- 各 variant の構造定義
- 右ペイン: TeX クエリ表示

**参照**: ABC395-D（3 種クエリ）、ABC380-E（2 種クエリ）

### 1.5 複数テストケース

**入力形式**:
```
T
N
A_1 A_2 ... A_N
（T ケース分繰り返し）
```

**制約**:
- 1 ≤ T ≤ 10^5
- 1 ≤ N ≤ 2×10^5
- ΣN ≤ 2×10^5
- 1 ≤ A_i ≤ 10^9

**E2E で検証する操作**:
- scalar 作成（T）
- 複数テストケーステンプレート適用
- ケース内で scalar N + 横配列 A を作成
- SumBound 追加（ΣN ≤ 2×10^5）
- 右ペイン: 複数ケース sample 生成

**参照**: 典型マルチケース問題

## 3. カテゴリ別の AST 対応状況

`doc/survey/atcoder-coverage-extended.md` の分析結果に基づく。

| カテゴリ | AST 表現 | 対応状況 | 備考 |
|---------|---------|---------|------|
| 基本配列 | Sequence[Scalar, Array] | ✅ 完全対応 | 最頻出パターン |
| グリッド | Sequence[Tuple, Repeat(String)] | ✅ 完全対応 | Grid テンプレートが必要 |
| 木入力 | Sequence[Scalar, Repeat(Tuple)] | ⚠️ | count = N-1 に Gap A |
| クエリ列 | Sequence[Tuple, Repeat(Choice)] | ⚠️ | Choice の Repeat 内ネスト未検証 |
| 複数テストケース | MultiTestCase wrapper | ✅ 完全対応 | IntroduceMultiTestCase Action あり |
